# ANDRAX 2.0 App — Build Notes

`android-app/` is now a **complete, buildable single-module Gradle project**.
The root of the module is `android-app/` itself (there is no separate `:app`
subproject). `./gradlew assembleDebug` produces an installable APK.

## Project layout

```
android-app/
├── build.gradle.kts              # Android application module (AGP 8.5.2, Kotlin 1.9.24)
├── settings.gradle.kts           # repositories + rootProject name
├── gradle.properties             # AndroidX, non-transitive R, JVM args
├── proguard-rules.pro            # minimal (minify disabled)
├── keystore.properties.example   # copy → keystore.properties for local release signing
├── gradlew / gradlew.bat         # Gradle wrapper (pinned to Gradle 8.9)
├── gradle/wrapper/…              # wrapper jar + properties
└── src/main/
    ├── AndroidManifest.xml
    ├── assets/
    │   ├── tool_registry.json        # copy of the backend registry (app catalog)
    │   └── workflow_registry.json    # generated by tools/build_workflow_registry.sh
    ├── java/com/andrax/two/…         # activities, data, launcher, model
    └── res/
        ├── values/themes.xml         # Theme.Andrax
        ├── values/strings.xml        # app_name
        ├── values/colors.xml         # icon + accent colors
        ├── drawable/ic_launcher_*.xml    # adaptive-icon layers (all vector)
        └── mipmap-anydpi-v26/ic_launcher*.xml
```

The launcher icon is an **all-vector adaptive icon**. Because `minSdk = 26`,
every supported device uses adaptive icons, so no density-specific PNGs are
needed.

## Requirements

* JDK 17 (JDK 21 also works to run Gradle).
* Android SDK with `platforms;android-34` and `build-tools;34.0.0`. Point Gradle
  at it via `ANDROID_HOME`/`ANDROID_SDK_ROOT`, or a git-ignored
  `android-app/local.properties`:
  ```properties
  sdk.dir=/path/to/Android/sdk
  ```

## Build & install

```sh
cd android-app
./gradlew assembleDebug       # → build/outputs/apk/debug/andrax-2.0-app-debug.apk
./gradlew installDebug        # onto the device that also runs Termux
```

## Release build & signing

The `release` build type is signed **only when signing credentials are
available** (so unattended `assembleRelease` still works and yields an unsigned
APK). Credentials are read from environment variables first (CI), then a local
`keystore.properties`. See [../docs/dev/05-signing-pipeline.md](../docs/dev/05-signing-pipeline.md).

```sh
cp keystore.properties.example keystore.properties   # then fill in real values
./gradlew assembleRelease     # → build/outputs/apk/release/andrax-2.0-app-release.apk (signed)
```

Verify:

```sh
"$ANDROID_HOME"/build-tools/34.0.0/apksigner verify --print-certs \
  build/outputs/apk/release/andrax-2.0-app-release.apk
```

## CI

`.github/workflows/ci.yml` builds the debug APK on every push/PR (the `app`
job), and `.github/workflows/release.yml` builds + signs + publishes the APK on
a `v*` tag. See [../docs/dev/04-cicd-pipeline.md](../docs/dev/04-cicd-pipeline.md).

## Sync the catalog after backend changes

```sh
bash ../tools/build_registry.sh            # sync app tool_registry.json from canonical + validate
bash ../tools/build_workflow_registry.sh   # regenerate workflow_registry.json
```

## Optional: Jetpack Compose version of ToolDetail

The activities are currently built programmatically to stay dependency-light.
If you prefer Compose, the detail screen collapses to:

```kotlin
@Composable
fun ToolDetail(tool: Tool, onRun: (List<String>) -> Unit) {
    var args by remember { mutableStateOf("") }
    Column(Modifier.padding(16.dp)) {
        Text(tool.name, style = MaterialTheme.typography.headlineSmall)
        Text(tool.description)
        Spacer(Modifier.height(8.dp))
        Text("andrax ${tool.example}", style = MaterialTheme.typography.bodySmall)
        OutlinedTextField(args, { args = it }, label = { Text("Arguments") })
        Button(onClick = { onRun(args.trim().split(Regex("\\s+")).filter { it.isNotEmpty() }) }) {
            Text("Run in Termux")
        }
    }
}
```

`onRun` would call `TermuxLauncher.runTool(context, tool.id, it)`. Adding Compose
means adding the Compose BOM + `compose` build feature to `build.gradle.kts`.
